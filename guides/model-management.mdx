## Model Configuration

### Global Default Models

Set default models for all new agents:

<Steps>
  <Step title="Open Settings">
    Profile → **Settings** → **Model Management**
  </Step>

  <Step title="Stream Agent Default">
    Choose default model for Stream agents
  </Step>

  <Step title="Iterative Agent Default">
    Choose default model for Iterative agents
  </Step>

  <Step title="Save">
    New agents will use these defaults
  </Step>
</Steps>

<Info>
  Existing custom agents keep their configured models. Only affects newly created agents.
</Info>

### Per-Agent Models

Customize models for individual agents:

<Steps>
  <Step title="Open Library">
    Go to **Library** → **Agents**
  </Step>

  <Step title="Edit Agent">
    Click **Edit** on an open-source agent
  </Step>

  <Step title="Select Model">
    Choose from available models
  </Step>

  <Step title="Save">
    Agent now uses the new model
  </Step>

  <Step title="Test">
    Try the agent with new model
  </Step>
</Steps>

<Warning>
  You can only change models for open-source agents you own. Closed-source agents have fixed models.
</Warning>

## Model Parameters

### Temperature

Controls randomness and creativity:

<Tabs>
  <Tab title="Low (0.0 - 0.3)">
    **Deterministic and focused**

    - More predictable output
    - Consistent code style
    - Less creative
    - Repeatable results

    **Best for:**
    - Production code
    - Bug fixes
    - Refactoring
    - Documentation

    **Example:** 0.2
  </Tab>

  <Tab title="Medium (0.4 - 0.7)">
    **Balanced**

    - Good creativity
    - Still consistent
    - Varied approaches
    - Reliable quality

    **Best for:**
    - General development
    - New features
    - UI components
    - Most use cases

    **Example:** 0.5 (default)
  </Tab>

  <Tab title="High (0.8 - 1.0)">
    **Creative and varied**

    - Very creative solutions
    - Diverse approaches
    - Less predictable
    - Experimental

    **Best for:**
    - Brainstorming
    - Design exploration
    - Multiple alternatives
    - Creative solutions

    **Example:** 0.9
  </Tab>
</Tabs>

### Max Tokens

Limits response length:

- **Stream Agents**: 4000-8000 tokens typical
- **Iterative Agents**: 2000-4000 per step
- **Higher**: More complete responses, higher cost
- **Lower**: Faster, cheaper, may truncate

<Info>
  1 token ≈ 4 characters. A component with 100 lines ≈ 400-800 tokens.
</Info>

### Other Parameters

<AccordionGroup>
  <Accordion icon="percentage" title="Top P (Nucleus Sampling)">
    Alternative to temperature:
    - **0.1**: Very focused
    - **0.5**: Balanced
    - **0.9**: Diverse
    - **1.0**: All possibilities

    Default: 1.0
  </Accordion>

  <Accordion icon="ban" title="Frequency Penalty">
    Reduces repetition:
    - **0.0**: No penalty (default)
    - **0.5**: Some reduction
    - **2.0**: Maximum reduction

    Higher = less repetitive code
  </Accordion>

  <Accordion icon="hashtag" title="Presence Penalty">
    Encourages new topics:
    - **0.0**: No penalty (default)
    - **0.5**: Moderate encouragement
    - **2.0**: Maximum encouragement

    Higher = more topic diversity
  </Accordion>
</AccordionGroup>

## Cost Management

### Understanding Pricing

AI model costs are based on tokens:

<Tabs>
  <Tab title="Input Tokens">
    **What you send**
    - Your prompts
    - System messages
    - Context/history
    - Code being edited

    Usually cheaper than output
  </Tab>

  <Tab title="Output Tokens">
    **What AI generates**
    - Generated code
    - Explanations
    - Responses

    Usually more expensive
  </Tab>
</Tabs>

### Model Cost Comparison

| Model | Input | Output | Relative Cost |
|-------|-------|--------|---------------|
| GPT-4 Turbo | $$$ | $$$ | High |
| Claude Opus | $$$ | $$$ | High |
| GPT-3.5 | $ | $ | Low |
| Claude Haiku | $ | $ | Low |
| Qwen 32B | $ | $ | Very Low |

<Info>
  Exact pricing varies - check provider websites for current rates. Tesslate doesn't mark up model costs.
</Info>

### Reducing Costs

<AccordionGroup>
  <Accordion icon="robot" title="Use Appropriate Models">
    - GPT-3.5/Haiku for UI
    - GPT-4/Opus for complex logic
    - Match model to task complexity
    - Don't use premium for simple tasks
  </Accordion>

  <Accordion icon="comments" title="Optimize Prompts">
    - Be specific and concise
    - Avoid unnecessary context
    - Clear, direct requests
    - Reduce back-and-forth
  </Accordion>

  <Accordion icon="history" title="Clear Context">
    - Clear chat history when changing topics
    - Don't carry unnecessary context
    - Start fresh for new features
    - Reduce token usage
  </Accordion>

  <Accordion icon="sliders" title="Adjust Parameters">
    - Lower max_tokens when appropriate
    - Use temperature 0.3 for predictable tasks
    - Disable verbose explanations
    - Request concise responses
  </Accordion>
</AccordionGroup>

## Switching Models

You can switch models anytime:

<Steps>
  <Step title="Library → Agents">
    Find the agent to update
  </Step>

  <Step title="Edit Agent">
    Click Edit (open-source only)
  </Step>

  <Step title="Change Model">
    Select new model from dropdown
  </Step>

  <Step title="Save">
    Changes apply immediately
  </Step>

  <Step title="Test">
    Try agent with new model
  </Step>
</Steps>

<Info>
  Each agent can use a different model. Mix and match based on needs.
</Info>
