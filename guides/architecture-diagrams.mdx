## Overview

Architecture Diagrams help you visualize and understand your project's structure, components, and data flow using AI-generated visual representations.

## Diagram Types

<Tabs>
  <Tab title="Component Tree">
    **Visualize React component hierarchy**

    Shows:
    - Parent-child relationships
    - Component dependencies
    - Prop flow
    - State management connections

    Best for: Understanding component structure
  </Tab>

  <Tab title="Data Flow">
    **Track how data moves through your app**

    Shows:
    - API calls and responses
    - State updates
    - Event handlers
    - Data transformations

    Best for: Understanding data management
  </Tab>

  <Tab title="File Structure">
    **Project organization diagram**

    Shows:
    - Folder hierarchy
    - File relationships
    - Import/export connections
    - Module dependencies

    Best for: Onboarding new team members
  </Tab>

  <Tab title="Custom">
    **Ask AI to generate specific diagrams**

    Examples:
    - "Show the authentication flow"
    - "Diagram the shopping cart feature"
    - "Visualize the API structure"

    Best for: Specific feature visualization
  </Tab>
</Tabs>

## Generating Diagrams

<Steps>
  <Step title="Open Project">
    Open the project you want to diagram
  </Step>

  <Step title="Click Diagram Icon">
    Find the Architecture Diagram button in toolbar
  </Step>

  <Step title="Choose Type">
    Select diagram type or use auto-detect
  </Step>

  <Step title="Generate">
    AI analyzes your codebase
  </Step>

  <Step title="View Result">
    Interactive diagram appears
  </Step>

  <Step title="Refine">
    Ask AI to adjust or add details
  </Step>
</Steps>

<Info>
  First diagram generation may take 30-60 seconds as AI analyzes your entire project.
</Info>

### Refactoring

Visualize before refactoring:

1. **Current State**: Generate diagram of existing structure
2. **Identify Issues**: Spot circular dependencies, complexity
3. **Plan Changes**: Sketch desired structure
4. **Refactor**: Make changes
5. **Verify**: Generate new diagram to confirm improvements

## AI Diagram Generation

### Natural Language Requests

Ask AI to generate specific diagrams:

**Examples:**
- "Show me how authentication works in this app"
- "Diagram the product catalog feature"
- "Visualize all components that use the user context"
- "Show the API endpoints and their relationships"
- "Create a flowchart of the checkout process"

### Refining Diagrams

<Tabs>
  <Tab title="Add Details">
    - "Include prop types in the component diagram"
    - "Show which components fetch data"
    - "Add state management connections"
  </Tab>

  <Tab title="Simplify">
    - "Remove utility files from the diagram"
    - "Show only the main page components"
    - "Hide third-party dependencies"
  </Tab>

  <Tab title="Focus">
    - "Focus on the shopping cart feature"
    - "Show only authentication-related components"
    - "Diagram the admin panel section"
  </Tab>

  <Tab title="Style">
    - "Use a vertical layout instead"
    - "Make the nodes larger"
    - "Use different colors for pages vs components"
  </Tab>
</Tabs>

## Exporting Diagrams

<Steps>
  <Step title="Open Diagram">
    View the diagram you want to export
  </Step>

  <Step title="Click Export">
    Click Export button in diagram toolbar
  </Step>

  <Step title="Choose Format">
    - PNG (for documents, presentations)
    - SVG (for web, scalable graphics)
    - JSON (for re-importing later)
  </Step>

  <Step title="Adjust Settings">
    Set resolution, size, background
  </Step>

  <Step title="Download">
    Save to your computer
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion icon="circle-xmark" title="Diagram Won't Generate">
    **Solutions:**
    - Ensure project has compiled successfully
    - Check for syntax errors in code
    - Verify project structure is valid
    - Try smaller scope (specific feature)
    - Restart and try again
  </Accordion>

  <Accordion icon="diagram-project" title="Diagram Too Complex">
    **Solutions:**
    - Filter to show specific parts
    - Generate feature-specific diagrams
    - Hide utilities and helpers
    - Use high-level view
    - Split into multiple diagrams
  </Accordion>

  <Accordion icon="triangle-exclamation" title="Missing Components">
    **Solutions:**
    - Ensure components are imported
    - Check file naming conventions
    - Verify exports are correct
    - Regenerate diagram
    - Report missing items
  </Accordion>

  <Accordion icon="arrows-rotate" title="Diagram Not Updating">
    **Solutions:**
    - Click refresh button
    - Close and regenerate
    - Clear cache
    - Check auto-update settings
  </Accordion>
</AccordionGroup>
