## Hot Module Replacement (HMR)

### How HMR Works

When you make changes:

<Steps>
  <Step title="Code Changes">
    You edit code or AI generates changes
  </Step>

  <Step title="Vite Detects">
    Vite development server detects file change
  </Step>

  <Step title="Module Updates">
    Only changed modules are replaced
  </Step>

  <Step title="State Preserved">
    Component state remains intact
  </Step>

  <Step title="Preview Updates">
    Changes appear instantly in preview
  </Step>
</Steps>

<Info>
  HMR is much faster than full page reloads and preserves your app state.
</Info>

### What Triggers HMR

- **Component changes**: Edit React components
- **Style updates**: Modify CSS files
- **Asset changes**: Update images or fonts
- **Config changes**: Some config updates (may need full reload)

## Troubleshooting

<AccordionGroup>
  <Accordion icon="circle-xmark" title="Preview Not Loading">
    **Problem:** Blank or loading preview

    **Solutions:**
    - Wait 30-60 seconds for initial startup
    - Check server status indicator
    - Restart development server in settings
    - Check browser console for errors
    - Refresh the preview
  </Accordion>

  <Accordion icon="ban" title="Changes Not Appearing">
    **Problem:** Code changes don't show in preview

    **Solutions:**
    - Check if HMR is working (console message)
    - Try manual refresh
    - Ensure files are saved
    - Restart dev server if needed
    - Check for syntax errors preventing build
  </Accordion>

  <Accordion icon="triangle-exclamation" title="Build Errors">
    **Problem:** Red error overlay in preview

    **Solutions:**
    - Read error message carefully
    - Check recent code changes
    - Look for syntax errors
    - Fix TypeScript errors
    - Ask AI agent to debug
  </Accordion>

  <Accordion icon="snail" title="Slow Preview">
    **Problem:** Preview is laggy or slow

    **Solutions:**
    - Close other browser tabs
    - Disable browser extensions
    - Check CPU usage
    - Optimize large images
    - Reduce component complexity
  </Accordion>
</AccordionGroup>
