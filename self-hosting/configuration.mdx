## Configuration File

All configuration is done through the `.env` file in the root directory. Start by copying the example:

```bash
cp .env.example .env
```

## Required Variables

These variables must be set for Tesslate Studio to function.

### SECRET_KEY

<ParamField path="SECRET_KEY" type="string" required>
  Secret key for JWT token signing and session encryption.
</ParamField>

**Generate:**
```bash
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

**Example:**
```env
SECRET_KEY=your-generated-secret-key-here
```

### LITELLM_MASTER_KEY

<ParamField path="LITELLM_MASTER_KEY" type="string" required>
  Master key for LiteLLM proxy authentication.
</ParamField>

**Generate:**
```bash
python -c "import secrets; print('sk-' + secrets.token_urlsafe(32))"
```

**Example:**
```env
LITELLM_MASTER_KEY=sk-your-litellm-master-key
```

### AI Provider API Keys

<Info>
  At least one AI provider API key is required for Tesslate Studio to generate code.
</Info>

<Tabs>
  <Tab title="OpenAI">
    ```env
    OPENAI_API_KEY=sk-your-openai-key
    ```
    Supports GPT-5, GPT-4, and GPT-3.5 models.
  </Tab>

  <Tab title="Anthropic">
    ```env
    ANTHROPIC_API_KEY=sk-your-anthropic-key
    ```
    Supports Claude 3.5 and Claude 3 models.
  </Tab>

  <Tab title="Google">
    ```env
    GOOGLE_API_KEY=your-google-api-key
    ```
    Supports Gemini Pro and Gemini Flash models.
  </Tab>

  <Tab title="Azure OpenAI">
    ```env
    AZURE_API_KEY=your-azure-key
    AZURE_API_BASE=https://your-resource.openai.azure.com
    AZURE_API_VERSION=2024-02-15-preview
    ```
  </Tab>
</Tabs>

## Optional Variables

### Application Settings

<ParamField path="APP_DOMAIN" type="string" default="studio.localhost">
  Domain where Tesslate Studio is hosted.
</ParamField>

```env
# Local development
APP_DOMAIN=studio.localhost

# Production
APP_DOMAIN=studio.yourcompany.com
```

<ParamField path="APP_PROTOCOL" type="string" default="http">
  Protocol for accessing the application. Options: `http`, `https`
</ParamField>

```env
# Local
APP_PROTOCOL=http

# Production with SSL
APP_PROTOCOL=https
```

<ParamField path="FRONTEND_URL" type="string" default="http://studio.localhost">
  Full URL where the frontend is accessible.
</ParamField>

```env
FRONTEND_URL=https://studio.yourcompany.com
```

### Database Configuration

<ParamField path="DATABASE_URL" type="string" default="postgresql+asyncpg://tesslate:tesslate_password@postgres:5432/tesslate_db">
  PostgreSQL connection string in asyncpg format.
</ParamField>

```env
# Docker default (no change needed)
DATABASE_URL=postgresql+asyncpg://tesslate:tesslate_password@postgres:5432/tesslate_db

# External managed database
DATABASE_URL=postgresql+asyncpg://admin:secure_pass@db.yourcompany.com:5432/tesslate
```

<ParamField path="POSTGRES_USER" type="string" default="tesslate">
  PostgreSQL username (Docker only).
</ParamField>

<ParamField path="POSTGRES_PASSWORD" type="string" default="tesslate_password">
  PostgreSQL password (Docker only).
</ParamField>

<Warning>
  Change the default password in production environments!
</Warning>

<ParamField path="POSTGRES_DB" type="string" default="tesslate_db">
  PostgreSQL database name (Docker only).
</ParamField>

### LiteLLM Configuration

<ParamField path="LITELLM_DEFAULT_MODELS" type="string" default="gpt-5o-mini,claude-3-haiku,gemini-pro">
  Comma-separated list of default AI models.
</ParamField>

```env
# Use only GPT-5 models
LITELLM_DEFAULT_MODELS=gpt-5o,gpt-5o-mini

# Mix of providers
LITELLM_DEFAULT_MODELS=gpt-5o-mini,claude-3-haiku,gemini-flash
```

<ParamField path="LITELLM_INITIAL_BUDGET" type="number" default="10.0">
  Initial API budget per user in USD.
</ParamField>

```env
# $50 initial budget
LITELLM_INITIAL_BUDGET=50.0
```

<ParamField path="LITELLM_PROXY_URL" type="string" default="http://litellm:4000">
  LiteLLM proxy endpoint URL.
</ParamField>

```env
# External LiteLLM instance
LITELLM_PROXY_URL=http://litellm.yourcompany.com:4000
```

### Container Runtime

<ParamField path="CONTAINER_MODE" type="string" default="docker">
  Container orchestration system to use. Options: `docker`
</ParamField>

<ParamField path="DOCKER_SOCKET_PATH" type="string">
  Path to Docker socket for container management.

  **Default:** `/var/run/docker.sock` (Linux/Mac), `//./pipe/docker_engine` (Windows)
</ParamField>

```env
# Linux/Mac
DOCKER_SOCKET_PATH=/var/run/docker.sock

# Windows
DOCKER_SOCKET_PATH=//./pipe/docker_engine
```

### Auto-Seeding

<ParamField path="AUTO_SEED_DATABASE" type="boolean" default="true">
  Automatically seed database with agents and templates on startup.
</ParamField>

```env
# Enable auto-seeding (recommended)
AUTO_SEED_DATABASE=true

# Disable for manual control
AUTO_SEED_DATABASE=false
```

**What gets seeded:**
- 4 marketplace agents (Stream Builder, Full Stack Agent, etc.)
- 3 project templates (Next.js, Vite+React+FastAPI, Vite+React+Go)
- 6 open-source customizable agents

### GitHub Integration

<ParamField path="GITHUB_CLIENT_ID" type="string">
  GitHub OAuth app client ID. Required for GitHub integration.
</ParamField>

<ParamField path="GITHUB_CLIENT_SECRET" type="string">
  GitHub OAuth app client secret.
</ParamField>

<Info>
  Create OAuth app at https://github.com/settings/developers
</Info>

```env
GITHUB_CLIENT_ID=your-github-client-id
GITHUB_CLIENT_SECRET=your-github-client-secret
```

### Logging

<ParamField path="LOG_LEVEL" type="string" default="INFO">
  Application logging level. Options: `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`
</ParamField>

```env
# Development
LOG_LEVEL=DEBUG

# Production
LOG_LEVEL=INFO
```

## Production Configuration

Recommended settings for production deployments:

```env
# Security
SECRET_KEY=<strong-random-key>
LITELLM_MASTER_KEY=sk-<strong-random-key>

# Domain
APP_DOMAIN=studio.yourcompany.com
APP_PROTOCOL=https
FRONTEND_URL=https://studio.yourcompany.com

# Database (use managed service)
DATABASE_URL=postgresql+asyncpg://user:pass@managed-db.provider.com:5432/tesslate

# Logging
LOG_LEVEL=INFO

# Auto-seed (optional in production)
AUTO_SEED_DATABASE=false
```

## Local AI Models (Ollama)

To use local AI models with Ollama:

<Steps>
  <Step title="Install Ollama">
    Download from [ollama.ai/download](https://ollama.ai/download)
  </Step>

  <Step title="Pull a model">
    ```bash
    ollama pull llama2
    ```
  </Step>

  <Step title="Configure LiteLLM">
    ```env
    LITELLM_DEFAULT_MODELS=ollama/llama2
    ```
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Invalid SECRET_KEY">
    **Problem:** SECRET_KEY is missing or invalid.

    **Solution:** Generate a new key and update `.env`:
    ```bash
    python -c "import secrets; print(secrets.token_urlsafe(32))"
    ```
  </Accordion>

  <Accordion title="Database connection failed">
    **Problem:** PostgreSQL is not accessible.

    **Solution:**
    ```bash
    # Check if postgres container is running
    docker compose ps postgres

    # Restart database
    docker compose restart postgres

    # Check logs
    docker compose logs postgres
    ```
  </Accordion>

  <Accordion title="API key not found">
    **Problem:** Missing or invalid AI provider API key.

    **Solution:**
    1. Verify key in `.env` has no extra spaces
    2. Test key with provider directly
    3. Restart orchestrator:
    ```bash
    docker compose restart orchestrator
    ```
  </Accordion>
</AccordionGroup>
