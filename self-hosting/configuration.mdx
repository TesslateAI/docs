## Configuration File

All configuration is done through the `.env` file in the root directory. Start by copying the example:

```bash
cp .env.example .env
```

## Required Variables

These variables must be set for Tesslate Studio to function.

### SECRET_KEY

<ParamField path="SECRET_KEY" type="string" required>
  Secret key for JWT token signing and session encryption.
</ParamField>

**Generate:**
```bash
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

**Example:**
```env
SECRET_KEY=your-generated-secret-key-here
```

### LITELLM_MASTER_KEY

<ParamField path="LITELLM_MASTER_KEY" type="string" required>
  Master key for LiteLLM proxy authentication.
</ParamField>

**Generate:**
```bash
python -c "import secrets; print('sk-' + secrets.token_urlsafe(32))"
```

**Example:**
```env
LITELLM_MASTER_KEY=sk-your-litellm-master-key
```

### AI Provider API Keys

<Info>
  At least one AI provider API key is required for Tesslate Studio to generate code.
</Info>

<Tabs>
  <Tab title="OpenAI">
    ```env
    OPENAI_API_KEY=sk-your-openai-key
    ```
    Supports GPT-5, GPT-4, and GPT-3.5 models.
  </Tab>

  <Tab title="Anthropic">
    ```env
    ANTHROPIC_API_KEY=sk-your-anthropic-key
    ```
    Supports Claude 3.5 and Claude 3 models.
  </Tab>

  <Tab title="Google">
    ```env
    GOOGLE_API_KEY=your-google-api-key
    ```
    Supports Gemini Pro and Gemini Flash models.
  </Tab>

  <Tab title="Azure OpenAI">
    ```env
    AZURE_API_KEY=your-azure-key
    AZURE_API_BASE=https://your-resource.openai.azure.com
    AZURE_API_VERSION=2024-02-15-preview
    ```
  </Tab>
</Tabs>

## Optional Variables

### Application Settings

<ParamField path="APP_DOMAIN" type="string" default="studio.localhost">
  Domain where Tesslate Studio is hosted.
</ParamField>

```env
# Local development
APP_DOMAIN=studio.localhost

# Production
APP_DOMAIN=studio.yourcompany.com
```

<ParamField path="APP_PROTOCOL" type="string" default="http">
  Protocol for accessing the application. Options: `http`, `https`
</ParamField>

```env
# Local
APP_PROTOCOL=http

# Production with SSL
APP_PROTOCOL=https
```

<ParamField path="FRONTEND_URL" type="string" default="http://studio.localhost">
  Full URL where the frontend is accessible.
</ParamField>

```env
FRONTEND_URL=https://studio.yourcompany.com
```

### Database Configuration

<ParamField path="DATABASE_URL" type="string" default="postgresql+asyncpg://tesslate:tesslate_password@postgres:5432/tesslate_db">
  PostgreSQL connection string in asyncpg format.
</ParamField>

```env
# Docker default (no change needed)
DATABASE_URL=postgresql+asyncpg://tesslate:tesslate_password@postgres:5432/tesslate_db

# External managed database
DATABASE_URL=postgresql+asyncpg://admin:secure_pass@db.yourcompany.com:5432/tesslate
```

<ParamField path="POSTGRES_USER" type="string" default="tesslate">
  PostgreSQL username (Docker only).
</ParamField>

<ParamField path="POSTGRES_PASSWORD" type="string" default="tesslate_password">
  PostgreSQL password (Docker only).
</ParamField>

<Warning>
  Change the default password in production environments!
</Warning>

<ParamField path="POSTGRES_DB" type="string" default="tesslate_db">
  PostgreSQL database name (Docker only).
</ParamField>

### LiteLLM Configuration

<ParamField path="LITELLM_DEFAULT_MODELS" type="string" default="gpt-5o-mini,claude-3-haiku,gemini-pro">
  Comma-separated list of default AI models.
</ParamField>

```env
# Use only GPT-5 models
LITELLM_DEFAULT_MODELS=gpt-5o,gpt-5o-mini

# Mix of providers
LITELLM_DEFAULT_MODELS=gpt-5o-mini,claude-3-haiku,gemini-flash
```

<ParamField path="LITELLM_INITIAL_BUDGET" type="number" default="10.0">
  Initial API budget per user in USD.
</ParamField>

```env
# $50 initial budget
LITELLM_INITIAL_BUDGET=50.0
```

<ParamField path="LITELLM_PROXY_URL" type="string" default="http://litellm:4000">
  LiteLLM proxy endpoint URL.
</ParamField>

```env
# External LiteLLM instance
LITELLM_PROXY_URL=http://litellm.yourcompany.com:4000
```

### Deployment Mode

<ParamField path="DEPLOYMENT_MODE" type="string" default="docker">
  Container orchestration system to use. Options: `docker`, `kubernetes`
</ParamField>

```env
# Local development with Docker + Traefik
DEPLOYMENT_MODE=docker

# Production with Kubernetes
DEPLOYMENT_MODE=kubernetes
```

<Info>
  Docker mode uses Docker Compose with Traefik for subdomain routing. Kubernetes mode uses K8s with NGINX Ingress and supports auto-scaling.
</Info>

### Application Domain

<ParamField path="APP_DOMAIN" type="string" default="localhost">
  Domain where Tesslate Studio is hosted (without protocol).
</ParamField>

<ParamField path="APP_BASE_URL" type="string">
  Full URL with protocol for OAuth redirects.
</ParamField>

```env
# Local development
APP_DOMAIN=localhost
APP_BASE_URL=http://localhost

# Production
APP_DOMAIN=studio.yourcompany.com
APP_BASE_URL=https://studio.yourcompany.com
```

### Auto-Seeding

<ParamField path="AUTO_SEED_DATABASE" type="boolean" default="true">
  Automatically seed database with agents and templates on startup.
</ParamField>

```env
# Enable auto-seeding (recommended)
AUTO_SEED_DATABASE=true

# Disable for manual control
AUTO_SEED_DATABASE=false
```

**What gets seeded:**
- 4 marketplace agents (Stream Builder, Full Stack Agent, etc.)
- 3 project templates (Next.js, Vite+React+FastAPI, Vite+React+Go)
- 6 open-source customizable agents

### OAuth Configuration

Configure OAuth providers for social login and integrations.

<Tabs>
  <Tab title="GitHub">
    <ParamField path="GITHUB_CLIENT_ID" type="string">
      GitHub OAuth app client ID.
    </ParamField>

    <ParamField path="GITHUB_CLIENT_SECRET" type="string">
      GitHub OAuth app client secret.
    </ParamField>

    <ParamField path="GITHUB_OAUTH_REDIRECT_URI" type="string">
      Callback URL for GitHub OAuth.
    </ParamField>

    ```env
    GITHUB_CLIENT_ID=your-github-client-id
    GITHUB_CLIENT_SECRET=your-github-client-secret
    GITHUB_OAUTH_REDIRECT_URI=https://studio.yourcompany.com/auth/github/callback
    ```

    <Info>
      Create OAuth app at https://github.com/settings/developers
    </Info>
  </Tab>

  <Tab title="Google">
    <ParamField path="GOOGLE_CLIENT_ID" type="string">
      Google OAuth client ID.
    </ParamField>

    <ParamField path="GOOGLE_CLIENT_SECRET" type="string">
      Google OAuth client secret.
    </ParamField>

    <ParamField path="GOOGLE_OAUTH_REDIRECT_URI" type="string">
      Callback URL for Google OAuth.
    </ParamField>

    ```env
    GOOGLE_CLIENT_ID=your-google-client-id.apps.googleusercontent.com
    GOOGLE_CLIENT_SECRET=your-google-client-secret
    GOOGLE_OAUTH_REDIRECT_URI=https://studio.yourcompany.com/auth/google/callback
    ```

    <Info>
      Create OAuth credentials at https://console.cloud.google.com/apis/credentials
    </Info>
  </Tab>
</Tabs>

### S3 Storage (Kubernetes Only)

Required for Kubernetes deployments using the S3 Sandwich pattern.

<ParamField path="S3_ACCESS_KEY_ID" type="string">
  S3 access key ID.
</ParamField>

<ParamField path="S3_SECRET_ACCESS_KEY" type="string">
  S3 secret access key.
</ParamField>

<ParamField path="S3_BUCKET_NAME" type="string" default="tesslate-projects">
  S3 bucket name for project storage.
</ParamField>

<ParamField path="S3_ENDPOINT_URL" type="string">
  S3 endpoint URL. Leave empty for AWS S3, set for MinIO or DigitalOcean Spaces.
</ParamField>

<ParamField path="S3_REGION" type="string" default="us-east-1">
  S3 region for signature calculation.
</ParamField>

```env
# AWS S3
S3_ACCESS_KEY_ID=AKIAIOSFODNN7EXAMPLE
S3_SECRET_ACCESS_KEY=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
S3_BUCKET_NAME=tesslate-projects
S3_REGION=us-east-1

# DigitalOcean Spaces
S3_ENDPOINT_URL=https://nyc3.digitaloceanspaces.com
S3_BUCKET_NAME=your-space-name
S3_REGION=nyc3

# MinIO (local development)
S3_ENDPOINT_URL=http://minio:9000
S3_BUCKET_NAME=tesslate-projects
```

### Kubernetes Settings

Configuration for Kubernetes deployments.

<ParamField path="K8S_DEVSERVER_IMAGE" type="string">
  Docker image for user project containers.
</ParamField>

<ParamField path="K8S_IMAGE_PULL_SECRET" type="string">
  Kubernetes secret name for private registry authentication.
</ParamField>

<ParamField path="K8S_STORAGE_CLASS" type="string" default="tesslate-block-storage">
  Kubernetes storage class for PVCs.
</ParamField>

<ParamField path="K8S_USE_S3_STORAGE" type="boolean" default="true">
  Enable S3 Sandwich pattern for project storage.
</ParamField>

```env
# Production (DigitalOcean)
K8S_DEVSERVER_IMAGE=registry.digitalocean.com/your-registry/tesslate-devserver:latest
K8S_IMAGE_PULL_SECRET=your-registry-secret
K8S_STORAGE_CLASS=do-block-storage

# AWS EKS
K8S_DEVSERVER_IMAGE=123456789.dkr.ecr.us-east-1.amazonaws.com/tesslate-devserver:latest
K8S_IMAGE_PULL_SECRET=ecr-credentials
K8S_STORAGE_CLASS=gp3

# Minikube (local)
K8S_DEVSERVER_IMAGE=tesslate-devserver:latest
K8S_IMAGE_PULL_SECRET=
K8S_STORAGE_CLASS=standard
```

Recommended settings for production deployments:

```env
# Security
SECRET_KEY=<strong-random-key>
LITELLM_MASTER_KEY=sk-<strong-random-key>

# Domain
APP_DOMAIN=studio.yourcompany.com
APP_PROTOCOL=https

# Auto-seed (optional in production)
AUTO_SEED_DATABASE=false
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Invalid SECRET_KEY">
    **Problem:** SECRET_KEY is missing or invalid.

    **Solution:** Generate a new key and update `.env`:
    ```bash
    python -c "import secrets; print(secrets.token_urlsafe(32))"
    ```
  </Accordion>

  <Accordion title="Database connection failed">
    **Problem:** PostgreSQL is not accessible.

    **Solution:**
    ```bash
    # Check if postgres container is running
    docker compose ps postgres

    # Restart database
    docker compose restart postgres

    # Check logs
    docker compose logs postgres
    ```
  </Accordion>

  <Accordion title="API key not found">
    **Problem:** Missing or invalid AI provider API key.

    **Solution:**
    1. Verify key in `.env` has no extra spaces
    2. Test key with provider directly
    3. Restart orchestrator:
    ```bash
    docker compose restart orchestrator
    ```
  </Accordion>
</AccordionGroup>
